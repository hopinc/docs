You can interface with Hop's API either over REST, or via libraries for certain
languages. Below is a list of our official, supported libraries.

### Client Side

| Name           | Language   | URL                                                                              | Official | Documentation                            |
| -------------- | ---------- | -------------------------------------------------------------------------------- | -------- | ---------------------------------------- |
| @onehop/client | TypeScript | [npmjs.com/package/@onehop/client](https://www.npmjs.com/package/@onehop/client) | ✅       | [/sdks/client/js](/sdks/client/js)       |
| @onehop/react  | TypeScript | [npmjs.com/package/@onehop/client](https://www.npmjs.com/package/@onehop/react)  | ✅       | [/sdks/client/react](/sdks/client/react) |

### Server Side

| Name       | Language   | URL                                                                      | Official | Documentation                              |
| ---------- | ---------- | ------------------------------------------------------------------------ | -------- | ------------------------------------------ |
| @onehop/js | TypeScript | [npmjs.com/package/@onehop/js](https://www.npmjs.com/package/@onehop/js) | ✅       | [/sdks/server/js](/sdks/server/js)         |
| hop-go     | Golang     | [github.com/hopinc/go](https://github.com/hopinc/go)                     | ✅       | [/sdks/server/go-api](/sdks/server/go-api) |
| hop-rs     | Rust       | [github.com/hopinc/rs](https://github.com/hopinc/rs)                     | ✅       | [/sdks/server/rs-api](/sdks/server/rs-api) |

### REST API

We don't have SDK support for every language out there - _yet_. So, if you'd
like to use our REST API directly, we have documentation on how to use that
[here](/reference/rest-api).
